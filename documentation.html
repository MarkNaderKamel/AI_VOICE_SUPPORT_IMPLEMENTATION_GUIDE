Laravel / PHP Implementation

Step 1: Create the Controller

Create app/Http/Controllers/Front/AiVoiceSupportController.php:

AiVoiceSupportController.php 
<?php

namespace App\Http\Controllers\Front;

use App\Http\Controllers\Controller;
use Illuminate\Http\Request;

class AiVoiceSupportController extends Controller
{
    /**
     * Get configuration for Gemini Live API
     * Returns API key for client-side WebSocket connection
     */
    public function getConfig(Request $request)
    {
        $apiKey = env('GEMINI_API_KEY');
        
        if (!$apiKey) {
            return response()->json([
                'success' => false,
                'error' => 'AI service not configured'
            ], 500);
        }

        return response()->json([
            'success' => true,
            'apiKey' => $apiKey,
            'model' => 'gemini-2.5-flash-native-audio-preview-12-2025',
            'voiceName' => 'Zephyr',
            'systemInstruction' => $this->getSystemPrompt()
        ]);
    }

    private function getSystemPrompt()
    {
        return "You are an AI voice assistant for our company. 
Be conversational, helpful, and concise. 
Respond naturally as if on a phone call.";
    }
}

React Implementation

React Component with @google/genai SDK

VoiceSupport.tsx 
import { useState, useRef } from 'react';
import { GoogleGenAI } from '@google/genai';

export function VoiceSupport() {
    const [isConnected, setIsConnected] = useState(false);
    const [status, setStatus] = useState('Click to start');
    const sessionRef = useRef(null);

    const startSession = async () => {
        try {
            setStatus('Connecting...');
            
            // Fetch config from your backend
            const res = await fetch('/api/ai-voice-support/config');
            const config = await res.json();
            
            const ai = new GoogleGenAI({ apiKey: config.apiKey });
            
            const session = await ai.live.connect({
                model: config.model,
                config: {
                    responseModalities: ['AUDIO'],
                    systemInstruction: config.systemInstruction,
                    speechConfig: {
                        voiceConfig: {
                            prebuiltVoiceConfig: {
                                voiceName: config.voiceName
                            }
                        }
                    }
                },
                callbacks: {
                    onopen: () => {
                        setIsConnected(true);
                        setStatus('Connected! Listening...');
                    },
                    onmessage: (msg) => handleAudio(msg),
                    onerror: (err) => setStatus('Error: ' + err.message)
                }
            });
            
            sessionRef.current = session;
            startAudioCapture(session);
            
        } catch (err) {
            setStatus('Failed to connect');
        }
    };

    return (
        <div className="voice-modal">
            <h2>Voice AI Support</h2>
            <p className="status">{status}</p>
            <button onClick={startSession}>
                {isConnected ? 'End Call' : 'Start Call'}
            </button>
        </div>
    );
}

Flutter Implementation

Dart Service Class

ai_voice_service.dart 
import 'dart:convert';
import 'package:http/http.dart' as http;
import 'package:web_socket_channel/web_socket_channel.dart';

class AiVoiceService {
  WebSocketChannel? _channel;
  String? _apiKey;
  
  Future<void> initialize() async {
    // Fetch config from backend
    final response = await http.get(
      Uri.parse('https://yoursite.com/api/ai-voice-support/config')
    );
    
    if (response.statusCode == 200) {
      final config = json.decode(response.body);
      _apiKey = config['apiKey'];
    }
  }
  
  Future<void> connect() async {
    if (_apiKey == null) await initialize();
    
    // Connect to Gemini Live API WebSocket
    final wsUrl = 'wss://generativelanguage.googleapis.com/ws/google.ai.generativelanguage.v1beta.GenerativeService.BidiGenerateContent?key=$_apiKey';
    
    _channel = WebSocketChannel.connect(Uri.parse(wsUrl));
    
    // Send setup message
    _channel!.sink.add(json.encode({
      'setup': {
        'model': 'models/gemini-2.5-flash-native-audio-preview-12-2025',
        'generationConfig': {
          'responseModalities': ['AUDIO'],
          'speechConfig': {
            'voiceConfig': {
              'prebuiltVoiceConfig': {'voiceName': 'Zephyr'}
            }
          }
        }
      }
    }));
    
    // Listen for responses
    _channel!.stream.listen((message) {
      // Handle audio response
      print('Received: $message');
    });
  }
  
  void disconnect() {
    _channel?.sink.close();
  }
}

.NET Implementation

ASP.NET Core Controller

AiVoiceController.cs 
using Microsoft.AspNetCore.Mvc;

[ApiController]
[Route("api/ai-voice-support")]
public class AiVoiceController : ControllerBase
{
    private readonly IConfiguration _configuration;

    public AiVoiceController(IConfiguration configuration)
    {
        _configuration = configuration;
    }

    [HttpGet("config")]
    public IActionResult GetConfig()
    {
        var apiKey = _configuration["GeminiApiKey"];
        
        if (string.IsNullOrEmpty(apiKey))
        {
            return StatusCode(500, new { 
                success = false, 
                error = "AI service not configured" 
            });
        }

        return Ok(new
        {
            success = true,
            apiKey = apiKey,
            model = "gemini-2.5-flash-native-audio-preview-12-2025",
            voiceName = "Zephyr",
            systemInstruction = @"You are an AI voice assistant. 
Be helpful and conversational."
        });
    }
}

Troubleshooting

Problem Solution
Connection closed immediately Check API key is valid and model name is correct
No audio output Ensure browser has audio permissions, check AudioContext is resumed
Microphone not working Use HTTPS in production, check browser mic permissions
404 on config endpoint Verify route is registered and server is running
WebSocket error Check network connectivity, firewall settings
Rate limit (429) Wait 60 seconds or upgrade to paid API tier

HTTPS Required

Microphone access requires HTTPS in production. Localhost works for development.

Complete Implementation Prompt 
I need you to implement an AI Voice Support feature in my Laravel/Web application. This feature allows users to have real-time voice conversations with an AI assistant powered by Google's Gemini Multimodal Live API.

## PROJECT REQUIREMENTS

### Technology Stack
- Backend: Laravel (PHP 7.4+)
- Frontend: HTML5, CSS3, JavaScript (Vanilla)
- AI API: Google Gemini Multimodal Live API (gemini-2.5-flash-native-audio-preview)
- Audio: Web Audio API for real-time audio streaming

### Features to Implement
- Real-time voice-to-voice conversations with AI
- Audio visualizer (canvas-based) showing audio waveforms
- Modal-based UI with modern dark theme
- Integration with custom knowledge base
- Secure API key handling (server-side only)
- Multiple voice options (Zephyr, Puck, Charon, Kore, Fenrir, Aoede)

## IMPLEMENTATION STEPS

### STEP 1: Create Laravel Controller
Create `app/Http/Controllers/Front/AiVoiceSupportController.php` with:
- `getConfig()` method returning: apiKey (from env), model name, voice name
- API key retrieved from `GEMINI_API_KEY` environment variable
- Rate limiting (10 requests/minute)

### STEP 2: Add Laravel Routes
Add to `routes/web.php`:
```php
Route::get('/api/ai-voice-support/config', 'Front\AiVoiceSupportController@getConfig');
Route::post('/api/ai-voice-support', 'Front\AiVoiceSupportController@query');
```

### STEP 3: Create JavaScript Module
Create `public/assets/front/js/ai-voice-support.js` with:
- Config object with getConfigEndpoint(), voiceName, model
- State management (session, stream, audioContext, status flags)
- Functions: init(), openModal(), closeModal(), startSession(), endSession()
- Audio handling: startAudioCapture(), handleAudioMessage(), playAudioChunk()
- Canvas-based audio visualizer

### STEP 4: Create CSS Styles
Create `public/assets/front/css/ai-voice-support.css` with:
- Modal overlay with backdrop blur
- Modal container with rounded corners
- Pulsing glow effect animation
- Canvas visualizer styles
- Control buttons (End Call, Retry, Close)
- Colors: Primary #f7931e, Background #1a1a2e

### STEP 5: Create Blade Template
Create `resources/views/front/partials/ai-voice-support.blade.php` with:
- Modal overlay, close button, header
- Status display, canvas visualizer
- Wave animation fallback, control buttons
- Contact link for human support

### STEP 6: Create Knowledge Base
Create `public/assets/front/js/ai-knowledge-base.js` with:
- documentContent: Company info as string
- company: { name, tagline, website, email, phone }
- services: Array of services with descriptions
- faq: Array of Q&A pairs
- systemPrompt: AI behavior instructions

### STEP 7: Environment Setup
Add to `.env`: GEMINI_API_KEY=your_api_key_here

### STEP 8: Include in Layout
Add to main layout before </body>:
```blade
@include('front.partials.ai-voice-support')
<link rel="stylesheet" href="{{ asset('assets/front/css/ai-voice-support.css') }}">
<script src="{{ asset('assets/front/js/ai-knowledge-base.js') }}"></script>
<script src="{{ asset('assets/front/js/ai-voice-support.js') }}"></script>
```

### STEP 9: Add Trigger
```html
<button onclick="openAIVoiceSupport()">AI Voice Support</button>
```

## SECURITY REQUIREMENTS
- NEVER hardcode API key in JavaScript
- Always retrieve from backend /config endpoint
- Use HTTPS in production
- Implement rate limiting
